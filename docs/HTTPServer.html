HTTPServer

staticHTTPServer.addCORSHandling(server, origin, headers, methods, maxAge)

http-server.js, line 194

Adds CORS handling to the Express server.

Name Type Description
server object

A server object (created via HTTPServer.create).
origin string

A value for the Access-Control-Allow-Origin header. If not provided, will be inferred by requests' Origin header.
headers string

A value for Access-Control-Allow-Headers header. If not provided, Content-Type will be used.
methods string

A value for Access-Control-Allow-Methods header. If not provided, only GET and POST methods will be allowed.
maxAge string

A value for Access-Control-Max-Age header. If not provided, one year will be used.

staticHTTPServer.execute(configurationRoot, main, prepare, root){Promise.<(void|Error)>}

http-server.js, line 43

Creates and executes a new HTTP server. It's basically a short hand for HTTPServer.create followed by HTTPServer.run.

Name Type Default Description
configurationRoot string

A root key in package.json holding application configuration. The notation is ROOT.HTTP, where ROOT is the top level key and HTTP is the part holding HTTP settings. The default is apes.httpServer.
main function null optional

A function that will define application routes. It will receive the server as the only argument.
prepare function null optional

A optional function that can setup additional middlewares or express behavior before routes definition. It will receive the server as the only argument.
root string null optional

The root directory of the application.
Returns:
Type Description
Promise.<(void|Error)> Nothing in case of success, the error otherwise.
